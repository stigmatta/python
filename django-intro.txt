--- Передумови ---
Управління пакетами: програма рір
Перевіряємо наявність та роботу:
у терміналі набираємо команду
> pip --version
pip 26.0.1 from C:\Python\Python313\Lib\site-packages\pip (python 3.13)

якщо команда не спрацьовує, то слід перевірити чи встановлений Python,
якщо немає - встановити, якщо є, то 
а) додати шлях до рір до системної змінної PATH
б) використовувати повний шлях для запуску рір 
   &"C:\Python\Python313\Scripts\pip" --version
 
Встановлюємо дозвіл на виконання сторонніх скриптів (Windows PowerShell)
PS> Set-ExecutionPolicy Unrestricted -Scope Process
Якщо виникають запитання, то відповісти (All - A)

--- Віртуальне оточення Python ---
Особливістю Python у схемі управління пакетами є те, що всі пакети 
встановлюються глобально. Це означає, що всі налаштування автоматично
поширюються на всі проєкти користувача. 
Переходимо до директорії, де буде створено проєкт (source/repos)
> python -m venv django
З'являється нова директорія (django) у якій є копія структури Python, 
що забезпечує локальність (оточення) встановлених пакетів.
Для роботи у локальному оточенні його необхідно активовувати
кожен раз, коли відновлюється робота з проєктом. 
Дана директорія є коренем проєкту, подальша робота відбувається у ній.
> cd django
Активуємо оточення (у залежності від інструментарію)
PS> ./Scripts/Activate.ps1
CMD> ./Scripts/activate.bat
MAC> source django/bin/activate
У результаті до командного промпту додається назва оточення
(django) PS ...>
!! При відновленні роботи з проєктом (після перезавантажень, 
вимкнень ПК) необхідно знов активувати оточення -- стежимо 
за промптом !!

---- Проєкт Django ----
Встановлюємо фреймворк 
(django)> python -m pip install Django
Цю команду бажано повторювати при клонуванні проєкту
ознакою успішного встановлення є поява управлінського 
скрипта django-admin.exe через який продовжується налаштування.
Його відсутність може свідчити про неправильну активацію 
оточення. 
Структура фреймворка дозволяє створювати декілька проєктів, У
рамках яких створювати декілька застосунків (applications)
Створюємо проєкт.
(django)> django-admin startproject project221
Додається директорія з назвою проєкту (project221) у якій - 
скрипт управління - manage.py

Коригуємо .gitignore так, щоб проєкт лишався у репозиторії, а 
оточення - ні
# Created by venv; see https://docs.python.org/3/library/venv.html
Include/*
Scripts/*
Lib/*
pyvenv.cfg

Створюємо застосунок
переходимо до проєкту 
(django)> cd project221
(django)> python manage.py startapp app221

Перевіряємо успішність встановлення запуском сервера
(django)> python manage.py runserver
- у браузері має відкриватись адреса, що зазначена за результатом запуску
- відбувається активація проєкту, зокрема, створюється БД (db.sqlite3)
Зупиняємо сервер.

---- Архітектура MVT ----
У Django використовується архітектура MVT (Model View Template)
Model - "постачальник" даних
Template - шаблон відображення - HTML документ з підстановкою виразів
View - функції, які забезпечують прийом запитів та підготовку
       відповідей (у т.ч. з шаблонами) - є аналогом контролерів

Маршрутизація: забезпечується співставленням шляху запиту та View 
за допомогою спеціальних файлів urls.py (початково такий файл є 
у проєкті). Слідуємо коментарям у цьому файлі (ідуть "з коробки")
у контексті п. "Including another URLconf"
- створюємо аналогічний файл у застосунку ./app221/urls.py
- оновлюємо імпорт у проєктному файлі (./project221/urls.py)
   from django.urls import include, path

- додаємо файл-конфігурацію до патернів URL 
   path('', include('app221.urls')),

- створюємо view: у файлі ./app221/views.py оголошуємо функцію hello
    from django.http import HttpResponse
    def hello(request):
        return HttpResponse("Hello, World!")

- у застосунку (./app221/urls.py) створюємо аналогічний масив
   з такою ж назвою, як у проєктного файлу 
    from django.urls import path
    from . import views
    urlpatterns = [
        path('', views.hello, name="hello"),
    ]

- реєструємо застосунок у проєкті: ./project221/settings.py
    INSTALLED_APPS = [
        ...
        'app221'   # додаємо цей рядок
    ]

- виконуємо міграції через зміни налаштувань 
   (зупиняємо сервер)
   (django)> python manage.py migrate

- запускаємо сервер
   (django)> python manage.py runserver
   та оновлюємо сторінку у браузері 


-----Templates -----
У директорії застосунку створюємо директорію templates. 
У ній створюємо файл index.html  (назва довільна)

Створюємо нову функцію view (index)
    def index(request):
        template = loader.get_template('index.html')
        return HttpResponse(template.render())
    
Реєструємо маршрут (юрл) - замінюємо hello на /hello, а 
index на ''
