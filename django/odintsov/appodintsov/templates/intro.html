Передумови

Управління пакетами: програма pip

Перевіряємо наявність та роботу — у терміналі набираємо команду:

> pip --version
pip 26.0.1 from C:\Python\Python313\Lib\site-packages\pip (python 3.13)

Якщо команда не спрацьовує, слід перевірити чи встановлений Python. Якщо немає — встановити. Якщо є:

  • додати шлях до pip до системної змінної PATH
  • або використовувати повний шлях для запуску pip: 
    &"C:\Python\Python313\Scripts\pip" --version

Встановлюємо дозвіл на виконання сторонніх скриптів (Windows PowerShell):

PS> Set-ExecutionPolicy Unrestricted -Scope Process

Якщо виникають запитання — відповісти All (A).

Віртуальне оточення Python

Особливістю Python у схемі управління пакетами є те, що всі пакети встановлюються глобально. Це означає, що всі налаштування автоматично поширюються на всі проєкти користувача.

Переходимо до директорії, де буде створено проєкт (source/repos):

> python -m venv django

З'являється нова директорія django, у якій є копія структури Python, що забезпечує локальність встановлених пакетів.

Для роботи у локальному оточенні його необхідно активовувати кожен раз, коли відновлюється робота з проєктом.

Переходимо до директорії:

> cd django

Активуємо оточення (залежно від інструментарію):

PS>  ./Scripts/Activate.ps1
CMD> ./Scripts/activate.bat
MAC> source django/bin/activate

У результаті до командного промпту додається назва оточення:

(django) PS ...>

Увага! При відновленні роботи з проєктом (після перезавантажень, вимкнень ПК) необхідно знову активувати оточення — стежимо за промптом!

Проєкт Django

Встановлюємо фреймворк:

(django)> python -m pip install Django

Цю команду бажано повторювати при клонуванні проєкту. Ознакою успішного встановлення є поява управлінського скрипта django-admin.exe, через який продовжується налаштування. Його відсутність може свідчити про неправильну активацію оточення.

Створюємо проєкт:

(django)> django-admin startproject project221

Додається директорія з назвою проєкту project221, у якій міститься скрипт управління manage.py.

Коригуємо .gitignore так, щоб проєкт лишався у репозиторії, а оточення — ні:

# Created by venv; see https://docs.python.org/3/library/venv.html
Include/*
Scripts/*
Lib/*
pyvenv.cfg

Створюємо застосунок — переходимо до проєкту:

(django)> cd project221
(django)> python manage.py startapp app221

Перевіряємо успішність встановлення запуском сервера:

(django)> python manage.py runserver
  • У браузері має відкриватись адреса, що зазначена за результатом запуску.
  • Відбувається активація проєкту, зокрема, створюється БД (db.sqlite3).

Зупиняємо сервер.

Архітектура MVT

У Django використовується архітектура MVT (Model View Template):

Model
«постачальник» даних
Template
шаблон відображення — HTML документ з підстановкою виразів
View
функції, які забезпечують прийом запитів та підготовку відповідей (у т.ч. з шаблонами) — є аналогом контролерів

Маршрутизація забезпечується співставленням шляху запиту та View за допомогою спеціальних файлів urls.py.

Слідуємо коментарям у файлі проєкту в контексті пункту "Including another URLconf":

  1. Створюємо аналогічний файл у застосунку ./app221/urls.py

  2. Оновлюємо імпорт у проєктному файлі ./project221/urls.py:

    from django.urls import include, path

  3. Додаємо файл-конфігурацію до патернів URL:

    path('', include('app221.urls')),

  4. Створюємо view — у файлі ./app221/views.py оголошуємо функцію hello:

    from django.http import HttpResponse

def hello(request):
    return HttpResponse("Hello, World!")

  5. У застосунку ./app221/urls.py створюємо масив urlpatterns:

    from django.urls import path
from . import views

urlpatterns = [
    path('', views.hello, name="hello"),
]

  6. Реєструємо застосунок у проєкті ./project221/settings.py:

    INSTALLED_APPS = [
    ...
    'app221'   # додаємо цей рядок
]

  7. Виконуємо міграції (зупиняємо сервер):

    (django)> python manage.py migrate

  8. Запускаємо сервер та оновлюємо сторінку у браузері:

    (django)> python manage.py runserver

Templates

У директорії застосунку створюємо директорію templates. У ній створюємо файл index.html (назва довільна).

Створюємо нову функцію view index:

def index(request):
    template = loader.get_template('index.html')
    return HttpResponse(template.render())

Реєструємо маршрут (URL) — замінюємо hello на /hello, а index на ''.
